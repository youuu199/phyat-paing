## Naming Convention Contract

### BEM Pattern (existing, preserve)

All existing class names use BEM: `block__element--modifier`. This pattern MUST continue for all new classes.

**Examples from codebase:**
- `bill-card__title`
- `bill-card__category--active`
- `auth-form__input`
- `modal-form__category-btn--active`

### New Classes for Inline Style Replacement

When converting inline styles to classes, use this naming pattern:
- Block: component name in kebab-case (`bill-card`, `bill-dashboard`)
- Element: descriptive noun (`header`, `flex-row`, `skeleton-row`)
- Modifier (if needed): state or variant (`--gap-sm`, `--justify-between`)

**Forbidden patterns:**
- No CSS Modules (`.module.css` suffix)
- No utility classes like Tailwind (`flex`, `gap-2`)
- No single-letter class names
- No inline styles (the entire point of this refactoring)

## Dark Mode Contract

### Selector Convention

All dark mode styles MUST use `[data-theme="dark"]` selector. No `.dark` class, no `.dark-mode`, no `prefers-color-scheme` media query.

**Existing pattern (preserve):**
```css
[data-theme="dark"] .component-name {
  /* dark overrides */
}
```

### New Dark Mode Tokens

Add these tokens to `index.css` `[data-theme="dark"]` block to replace hardcoded colors:

```css
[data-theme="dark"] {
  /* ... existing tokens ... */
  --color-danger-dark: #450a0a;
  --color-danger-border-dark: #dc2626;
  --color-warning-dark: #451a03;
  --color-on-accent: #fff;
}
```

Also add to `:root`:
```css
:root {
  /* ... existing tokens ... */
  --color-on-accent: #fff;
}
```

### Dark Mode Override Pattern

Each component's CSS file ends with a dark mode section:

```css
/* ComponentName.css */
/* ... component styles ... */

/* ---- Dark Mode ---- */
[data-theme="dark"] .component-name {
  background: var(--color-surface);
  color: var(--color-text);
}
```

No component may use hardcoded hex colors that differ between light and dark mode without a CSS variable.
