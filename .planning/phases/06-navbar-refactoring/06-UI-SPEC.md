# Navbar Refactoring — UI Design Contract

> Visual and interaction contract for extracting a shared NavItem component, improving mobile nav accessibility, and unifying brand display. Generated by gsd-ui-researcher.

---

## Problem Statement

The current navbar has two independent nav implementations:

1. **Desktop header** (`App.tsx` lines 81–93) — inline `<button>` loop over `NAV_ITEMS`
2. **Mobile slide-over** (`MobileNav.tsx` lines 82–92) — separate `<button>` loop over same `NAV_ITEMS`

Both render the same pattern (icon + translated label + active state) but with different CSS classes and no shared component. The mobile nav lacks a proper focus trap (Tab can escape the dialog). The brand name uses `t('app.brand')` in mobile but the desktop header uses `t('app.title')` (the full "Phyat Paing (ဖြတ်ပိုင်း)" with tagline).

## Component Contract

### 1. NavItem — NEW shared component

**File:** `client/src/components/NavItem.tsx`

**Props interface:**
```typescript
interface NavItemProps {
  page: Page;
  icon: LucideIcon;
  label: string;
  isActive: boolean;
  onClick: (page: Page) => void;
  variant: 'desktop' | 'mobile';
}
```

**Markup structure:**
```tsx
<button
  className={`nav-item nav-item--${variant}${isActive ? ' nav-item--active' : ''}`}
  onClick={() => onClick(page)}
  aria-current={isActive ? 'page' : undefined}
>
  <Icon size={variant === 'mobile' ? 18 : 16} strokeWidth={1.75} />
  {label}
</button>
```

**CSS classes:**
- Base: `nav-item` — shared layout (flex, align-items, gap, border-radius, cursor, transition)
- Desktop modifier: `nav-item--desktop` — `padding: var(--space-1) var(--space-3)`, `font-size: var(--text-sm)`, `min-height: auto`
- Mobile modifier: `nav-item--mobile` — `padding: var(--space-3)`, `font-size: var(--text-base)`, `min-height: 44px`, `width: 100%`
- Active modifier: `nav-item--active` — `background: var(--color-accent)`, `color: #fff`, `border-color: var(--color-accent)`
- Hover: `background: var(--color-surface-hover)`, `color: var(--color-text)`
- Active hover: `opacity: 0.9`, `color: #fff`
- Focus visible: `outline: 2px solid var(--color-accent)`, `outline-offset: 2px`

### 4. Mobile Nav Focus Trap — UPDATE

**File:** `client/src/components/MobileNav.tsx`

**Current state:** Focus moves to first nav item on open, Escape closes, but Tab can escape the dialog.

**Required behavior:**
- Trap focus within the dialog when open
- Tab/Shift+Tab cycles through: Close button → NavItem × 4 → ThemeToggle → Logout button
- Use a `useEffect` that attaches a `keydown` listener on the `navRef` element
- On `Tab`: if focus is at last interactive element, move to first; if `Shift+Tab` at first, move to last
- Restore focus to hamburger button on close (store ref or use `aria-controls` target)

**Focus order (mobile nav):**
1. Close button (`.mobile-nav__close`)
2. Dashboard NavItem
3. Analytics NavItem
4. Profile NavItem
5. Settings NavItem
6. ThemeToggle (`.theme-toggle`)
7. Logout button (`.mobile-nav__logout`)

## Interaction Spec

| Trigger | Behavior |
|---------|----------|
| Desktop nav item click | Sets active page, no other side effects |
| Desktop nav item keyboard | Enter/Space activates, Tab moves to next nav item |
| Mobile hamburger tap | Opens slide-over nav, focuses close button |
| Mobile nav item tap | Navigates to page, closes nav |
| Mobile close button tap | Closes nav, returns focus to hamburger |
| Mobile backdrop tap | Closes nav, returns focus to hamburger |
| Mobile Escape key | Closes nav, returns focus to hamburger |
| Mobile Tab (forward) | Cycles: close → nav items → theme → logout → close |
| Mobile Shift+Tab (backward) | Cycles in reverse |
| Body scroll | Locked (`overflow: hidden`) when mobile nav is open |

## Anti-Patterns to Avoid

- ❌ Do NOT duplicate nav item markup in both desktop and mobile — use shared `NavItem`
- ❌ Do NOT use `tabIndex={-1}` on nav items — all must be keyboard accessible
- ❌ Do NOT hardcode English text in nav — use `t(labelKey)` from `NAV_ITEMS`
- ❌ Do NOT add `onClick` without corresponding keyboard handler — use `<button>` elements
- ❌ Do NOT forget `aria-current="page"` on active nav item
- ❌ Do NOT use emoji in nav items — lucide-react icons only (Phase 5 decision)
