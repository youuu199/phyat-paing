# Hamburger Menu Update — UI Design Contract

> Visual and interaction contract for refining the existing hamburger menu + mobile navigation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Problem Statement

The current hamburger menu works but has several UX gaps:

1. **No backdrop overlay** — Content behind the nav remains visible and interactive when the menu is open
2. **Unconventional animation** — Slides from top (`translateY(-100%)`) instead of the expected right/left slide
3. **Hacky X animation** — Uses `translateY(7px)` offsets that don't cleanly form an X
4. **No notification badges** — Dashboard nav item doesn't show upcoming bill count
5. **No swipe-to-close** — Mobile users expect gesture-based dismissal
6. **Hamburger button isolation** — The button is nested deep in header markup, making its mobile positioning fragile

## Component Changes

### 1. Hamburger Button — `App.tsx` lines 15–29

**Current:** Inline `Hamburger` component with 3 `<span>` lines using `translateY(7px) rotate(45deg)` for X state.

**Updated:**
- Move hamburger to a dedicated `Hamburger.tsx` component file
- Fix X animation using `transform-origin: center` on each line:
  - Line 1: `transform: rotate(45deg)` (no translateY — use `top` line's transform-origin at center)
  - Line 2: `opacity: 0` (unchanged)
  - Line 3: `transform: rotate(-45deg)`
- Add `will-change: transform` for GPU acceleration
- ARIA attributes:
  - `aria-expanded={isOpen}`
  - `aria-controls="mobile-nav"`
  - `aria-label={isOpen ? "Close navigation menu" : "Open navigation menu"}`
- Spacing: `gap: var(--space-2)` (8px) — replaces current `5px` hack

### 2. MobileNav — `MobileNav.tsx`

**Current:** Full-screen overlay sliding from top with `translateY(-100%)`.

**Updated:**
- Change animation to **slide from right**: `transform: translateX(100%)` → `translateX(0)`
- Add **backdrop overlay** (`<div class="mobile-nav__backdrop">`) behind the nav panel:
  - Semi-transparent black (`rgba(0, 0, 0, 0.4)`)
  - Fades in with opacity transition
  - Click dismisses the nav
  - Covers full viewport, z-index below nav panel
- Add **notification badge** on Dashboard nav item:
  - Accept optional `upcomingCount` prop
  - Small pill badge (accent color, white text, `--text-xs`)
  - Positioned top-right of the nav item
- Add **swipe-to-close** gesture:
  - Track `touchstart` / `touchmove` / `touchend` on nav panel
  - If swipe-right distance > 80px, close the nav
  - Animate panel following finger during swipe (`translateX(dragX)`)
- Keep existing keyboard support (Escape to close, focus trap, body scroll lock)

### 3. CSS Changes — `App.css`

**Hamburger button (lines 2482–2533):**
- Remove `translateY(7px)` from X animation lines
- Use proper `transform-origin` per line for clean X formation
- Add `will-change: transform` to `.hamburger__line`

**Mobile nav (lines 2535–2679):**
- Change `.mobile-nav` from `translateY(-100%)` to `translateX(100%)`
- Change `.mobile-nav--open` from `translateY(0)` to `translateX(0)`
- Add `.mobile-nav__backdrop` styles:
  ```css
  .mobile-nav__backdrop {
    display: none;
    position: fixed;
    inset: 0;
    z-index: 299;
    background: rgba(0, 0, 0, 0.4);
    opacity: 0;
    transition: opacity var(--transition-smooth);
  }
  .mobile-nav__backdrop--visible {
    display: block;
    opacity: 1;
  }
  ```
- Add `.mobile-nav__badge` styles:
  ```css
  .mobile-nav__badge {
    display: inline-flex;
    align-items: center;
    justify-content: center;
    min-width: 22px;
    height: 22px;
    padding: 0 var(--space-2);
    border-radius: var(--radius-full);
    background: var(--color-accent);
    color: #fff;
    font-size: var(--text-xs);
    font-weight: var(--font-weight-bold);
    margin-left: auto;
  }
  ```
- Add swipe drag styles for `.mobile-nav--dragging` (inline `transform` override)

**Updated:**

## Interaction Spec

| Trigger | Behavior |
|---------|----------|
| Hamburger tap | Toggle mobile nav open/close, animate X transition |
| Backdrop tap | Close mobile nav |
| Nav item tap | Navigate to page, close nav |
| Escape key | Close mobile nav |
| Swipe right > 80px | Close mobile nav |
| Swipe right (partial) | Animate panel following finger, snap back if < 80px (200ms ease-out) |
| Body scroll | Locked when nav is open |

## Accessibility

- Hamburger: `aria-expanded={isOpen}`, `aria-controls="mobile-nav"`, `aria-label={isOpen ? "Close navigation menu" : "Open navigation menu"}`
- Close button: `aria-label="Close menu"`
- Mobile nav: `role="dialog"`, `aria-modal="true"`, `aria-label="Navigation menu"`
- Nav items: visible text labels ("Dashboard", "Profile", "Settings", "Logout")
- Active page: `aria-current="page"` on active nav item
- Focus: First nav item focused on open (after 200ms animation delay)
- Escape: Closes nav from any focused element
- Body scroll: Locked via `overflow: hidden` when open
- Badge: `aria-label="{count} upcoming bills"` for screen readers
- Logout: `window.confirm("Are you sure you want to log out?")` before action
